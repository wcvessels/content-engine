Getting Started with Claude Code: Install and First Use

Set up the three ways to use Claude, Desktop, the VS Code extension, and the terminal, then run your first real task on your own files.

content-engine (Will Vessels) · 2026-06-27

For: Anyone new to Claude who wants to move past copy and paste and use it directly on their files

Learning objectives

  • Tell apart the three ways to use Claude (Desktop, the VS Code extension, and the terminal) and pick the right one for a task.
  • Install Claude Desktop and confirm you downloaded the official app.
  • Install Claude Code by setting up Node.js and running one npm command, then sign in.
  • Add the Claude Code extension to VS Code and open Claude next to your files.
  • Run your first real task by having Claude summarize a file in a folder.

Beyond copy and paste

Most people use Claude like a search engine that writes better sentences. You type a question, you get an answer, and then you copy it and paste it somewhere else. That works, but it is like using a car to check the mailbox. The car can take you across the country; you just have not left the driveway yet.

This guide leaves the driveway. By the end you will have all three ways to use Claude installed and working, a clear sense of when to use which one, and your first real task run directly on your own files instead of through copy and paste.

NoteThe goal here is real deliverables sitting on your computer, not just a better understanding of the tools. Each step ends with something installed and working.

The three ways to use Claude

The thing most people miss is that these are not three different AIs. It is one model with three different ways to access it. The desktop app is the conversation layer, the VS Code extension is the editor layer, and the terminal is the command line layer. Same brain behind all three. The difference is what each interface lets that brain see and do.

Claude Desktop is the downloaded version of the chat you already know. It is great for conversation, planning, and thinking out ideas, but on its own it cannot see your files or run code; everything goes through copy, paste, or uploads. Claude Code in VS Code puts your files on the left and Claude on the right: it reads your project, edits files in place, and runs commands, so you describe what you need instead of pasting it. The terminal version is the same engine in a lighter wrapper, handy when your work is a folder of documents rather than a code project.

NoteYou may have heard of Cursor, Windsurf, Copilot, or Google's Anti-Gravity. Those are separate products built on top of VS Code, and some do not even use Claude. This guide covers Claude's own tools inside the Anthropic environment.

Which one to use when

The pattern is straightforward once you see it. The normal chat or desktop is for a good conversation and quick work you want to bust out. Claude Code is for action and scale: it is not much different on one file, but as soon as you have a hundred files to roll through, it is far easier. The terminal is the most flexible, since you can open it anywhere, though it is less visual and has a bit of a learning curve.

The part worth paying attention to is iterating on output. With plain chat you read the response, decide what is wrong, copy it, and start again. With Claude Code you can say "that is not right, fix this," and it changes the file in place, so you are delegating instead of running rounds of copy and paste. Pick the one that feels right for the work in front of you rather than forcing everything through a single interface.

TipWhen chat feels like it almost gets it but not quite, it is usually because it is working from a pasted excerpt instead of your actual project. The interface is the middleman. The fix is to give Claude direct access to your files.

Install Claude Desktop

Search for Claude Desktop or Download Claude and open the official download page. Make sure the result is actually from Anthropic at claude.com before you download anything. Then pick the build for your system: Mac, Windows, or ARM64.

TipNot sure whether you are on a standard or ARM processor? On Windows, open Settings, go to System, then About, and read the System type line. "x64-based processor" is the standard build; "ARM-based processor" means pick the ARM64 download.

Download the installer, run it, give it permissions, and log in exactly as you would anywhere else you use Claude. Once it is open, the desktop app can even help you install Claude Code for you, the non-manual way. The next sections show the manual path so you understand what is happening and can put Claude directly on your system.

Install Claude Code

Claude Code needs one thing first: Node.js. This is a free JavaScript runtime that lets the tool run on your computer. Go to nodejs.org and use the Get Node.js download (the LTS version is the safe choice). It is worth having installed anyway if you do any building.

With Node.js installed you get the npm command. Open your command line (type cmd on Windows, or the integrated terminal on Mac) and run one command to install Claude Code straight from Anthropic, then check that it worked:

  1. Install Node.js from nodejs.org (LTS version).
  2. Open your terminal.
  3. Run npm install -g @anthropic-ai/claude-code.
  4. Verify by typing claude and hitting enter.
NoteIf the command trips on packages, permissions, or where things landed on your computer, let Claude Desktop or Claude itself walk you through it. Setups vary machine to machine, and that is exactly the kind of thing it is good at fixing.

Sign in to Claude Code

Now you can type claude from anywhere on your computer. The first time, it asks whether you trust the current workspace; say yes. Then it asks you to log in.

TipThe easy path: choose log in with subscription. It opens a tab, you click authorize, and you are done. You can also paste an API key from the Anthropic console instead, but if you already pay for a subscription you do not need one, and the subscription login is simpler.

Add Claude Code to VS Code

VS Code (Visual Studio Code) is Microsoft's free editor. You do not need to know how to code to use it; it just lets you open any folder and see everything as a tree. Once Claude Code is installed from the step above, add the extension so Claude lives in the sidebar next to your files.

  1. Install VS Code (on Windows you can grab it from the App Store or the official site).
  2. Open VS Code and go to Extensions (the squares icon in the left sidebar, or press Ctrl+Shift+X).
  3. Search for Claude Code and install Claude Code for VS Code by Anthropic.
  4. Restart VS Code if it asks you to.
  5. Open a folder, trust it, and click the button to open Claude beside your files.
NoteOne thing that trips people up: when you open a brand-new folder and see nowhere to add anything, click on a file to open a page first. That gets the extension panel to show up.

Run your first task

Pick one thing from your own work and actually run it. Put a single file in a folder, then have Claude Code summarize it or answer questions about it. Try the same thing in Claude Desktop, and notice the difference in the workflow: in VS Code, Claude already knows which file you mean, finds it, and reads it, which is one less layer of work than uploading it by hand.

TipThe point is not just installing something that works. It is starting to think about how you build and scale: if you are running a project, a startup, or a content pipeline, which tool fits the way you actually work?

Where you are now

You now have three ways to use Claude installed and working: the desktop app for conversation, the VS Code extension for working directly in your files, and the terminal for quick, flexible jobs. Pick the one that fits each task rather than forcing everything through one. Get comfortable with one setup first; the habits carry straight over to the others.

NoteYou have left the driveway. The shift is from copy and paste to Claude working directly on your files. That is the whole point, and everything else builds on it.
