# Letterhead

The brand frame applied to every rendered deliverable: title-page layout, header/footer, logo placement, and the static disclaimer line. Loaded at **04-render** (brand is filled at render, never at layout). Colors and fonts come from `../design-system.md`; name/tagline/author come from `../identity.md`.

Shipped PRE-FILLED with the content-engine brand. `/setup` rewrites each `{{PLACEHOLDER}}`.

## Logo placement

- **Asset** ({{LOGO_PATH}}): `_config/brand/logo/wordmark.svg` (light), `wordmark-dark.svg` (dark), `mark.svg` (small/footer mark), `favicon-32.png` / `favicon-180.png`.
- **Title page:** an indigo+amber accent rule at the very top, then the mark + wordmark lockup top-left, then the eyebrow and title block. Implemented in `_skills/render-doc/render.py` (inline SVG mark + wordmark text, both self-contained).
- **HTML header:** same mark + wordmark lockup at the top of the title block; a base64 favicon is inlined into every page.
- **PDF running header:** none. The title page carries the logo; body pages stay clean (the static footer is the only running element). This keeps the reader's eye on content.

## Title-page layout (first page, PDF and HTML hero)

```
[====== indigo ======][ amber ]               <- accent rule, full width, 5px
[mark] content-engine                         <- lockup, top-left

{{HEADER_TEXT}}                               <- small, --secondary, eyebrow (doc_type)

<Document title>                              <- h1, --primary, --font-heading
<Subtitle: audience + what they'll do>        <- --secondary, --font-body

Prepared by {{AUTHOR}}  ·  {date}             <- small, --secondary
```

- **Eyebrow** ({{HEADER_TEXT}}): `content-engine` (the producing brand; small caps feel, --secondary).
- The doc title, subtitle, author, and date come from the doc-model `meta` block (filled from `identity.md`).
- `break-after: page` after the title page in print.

## Header / footer

- **Header** (body pages): none beyond the title page (see logo placement).
- **Footer** ({{FOOTER_TEXT}}): the static disclaimer line, repeated on every page in print, shown once at the document foot in HTML.

## The static disclaimer + confidentiality line (the only running footer)

- **Disclaimer** ({{DISCLAIMER}}): `Generated by content-engine from a recorded session. Verify against the source before relying on any step.`
- **Confidentiality** ({{CONFIDENTIALITY}}): `Internal use only.`

Rendered as one footer line, `--secondary`, `small`, centered:

> Internal use only.  ·  Generated by content-engine from a recorded session. Verify against the source before relying on any step.

Notes:
- This is the ONLY footer content. No page numbers, no print-only TOC (decision D4). The HTML sticky nav covers navigation.
- The confidentiality default is `Internal use only.` Change it per deliverable in the run brief if a document is meant to be public; the public sample ships with a public-appropriate line.
- The disclaimer is a trust signal, not legalese: it tells the reader this came from a recording and that the source is the final word, matching the fidelity posture.

## Fields the doc-model footer carries

The doc-model `footer` object (see `_standards/doc-model-schema.md`) holds `disclaimer` and `confidentiality`, both filled here at render. Reader-facing text only: no timestamps, no citations, no provenance ever appears in the footer.
